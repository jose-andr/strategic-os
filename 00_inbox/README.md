# Inbox

## Purpose

The inbox is the capture point for raw notes, ideas, prompts, questions, meeting fragments, opportunities and unfinished thoughts that have not yet been processed into the Strategic OS.

Use this folder to collect material before deciding where it belongs.

## What belongs here

- Raw notes
- Unsorted ideas
- Draft prompts
- Meeting fragments
- Questions to revisit
- Potential strategic opportunities
- Early reflections
- Unprocessed lessons
- Links or references to review
- Work that needs triage

## What does not belong here

- Final decision briefs
- Completed templates
- Long-term project artefacts
- Career evidence
- Sensitive organisational source data
- Customer-level records
- Governed datasets
- Confidential extracts

## Triage options

Inbox items should eventually move to one of these locations:

| If the item is... | Move or convert to... |
|---|---|
| A strategic recommendation | `03_decision_briefs/` |
| A possible future opportunity | `02_strategic_opportunities/` |
| A reusable lesson | `05_lessons_learned/` |
| A stakeholder pattern | `06_stakeholder_patterns/` |
| A project-specific artefact | `08_projects/` |
| A career proof point | `01_career/` |
| A reusable model or method | `04_frameworks/` |
| A public-facing idea | `09_thought_leadership/` |
| A reusable structure | `10_templates/` |

## Suggested naming

Use simple temporary names.

Examples:

- `2026-06-15_raw-note_channel-strategy.md`
- `2026-06-15_idea_chief-of-staff-agent.md`
- `2026-06-15_question_stakeholder-alignment.md`

## Review rhythm

Review the inbox weekly.

For each item, decide whether to:

- keep
- move
- convert
- merge
- delete
- archive

## Review checklist

Before moving an inbox item, check:

- [ ] Is the item still useful?
- [ ] Does it support a decision, opportunity, lesson, project or career outcome?
- [ ] Is the right destination folder clear?
- [ ] Is any sensitive information removed or generalised?
- [ ] Should this become a reusable pattern or template?
- [ ] Is there a clear next action?
