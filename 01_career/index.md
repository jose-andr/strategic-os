# Career Index

## Purpose

This index tracks career evidence, opportunities, feedback, promotion material and capability proof points stored in `01_career/`.

Use it to make career evidence easier to find, review and convert into role applications, promotion readiness material, portfolio examples, interview stories and opportunity positioning.

The index supports the Strategic OS North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

This is not a diary.

It is a lightweight index of evidence that may support future career decisions, strategic opportunities or reputation building.

## What To Index

Index career material that:

- demonstrates strategic influence
- shows organisational impact
- captures useful feedback
- supports promotion readiness
- supports a role application
- demonstrates systems thinking
- demonstrates human-centred or business design practice
- demonstrates AI-enabled capability
- shows stakeholder influence
- shows delivery and follow-through
- may become a portfolio example
- may become an interview story
- may support a future opportunity

Do not index:

- low-value reflections
- private employee matters
- confidential organisational material
- raw feedback that should not be stored
- sensitive stakeholder commentary
- vague claims without evidence
- material that cannot be safely reused

## Status Values

Use simple status values.

| Status | Meaning |
| --- | --- |
| Draft | Evidence is still being shaped |
| Active | Evidence is current and useful |
| Reusable | Evidence is strong enough to reuse in applications, interviews or reviews |
| Converted | Evidence has become a resume bullet, portfolio example, promotion proof point or opportunity note |
| Superseded | Replaced by stronger or more current evidence |
| Archived | Retained for history only |

## Evidence Types

Use one of these evidence types where possible.

| Evidence Type | Description |
| --- | --- |
| Achievement | Completed contribution or outcome |
| Feedback | Feedback or reputation signal |
| Opportunity | Possible role, project, secondment or strategic move |
| Promotion Proof Point | Evidence supporting readiness for progression |
| Capability Example | Evidence of a specific skill or practice |
| Portfolio Example | Work that may become a case study or interview story |
| Reputation Signal | Evidence that others associate the user with a desired capability |
| Learning Evidence | Lesson that shows growth, judgement or improved practice |

## Capability Categories

Use one or more capability categories.

| Capability | Description |
| --- | --- |
| Strategic Influence | Shaping direction, decisions, options or recommendations |
| Organisational Impact | Improving clarity, risk, process, alignment or outcomes |
| Customer-Centred Practice | Bringing customer needs into decisions |
| Systems Thinking | Making complexity, dependencies or handovers visible |
| Human-Centred Design | Using design methods to understand, frame or prototype |
| Business Design | Connecting desirability, feasibility, viability and value |
| AI-Enabled Capability | Using AI responsibly to improve strategic work |
| Stakeholder Influence | Building trust, alignment or shared understanding |
| Delivery And Follow-Through | Moving work from thinking into useful output |

## Field Guidance

## Date

Use the date the evidence was created, received or last meaningfully updated.

Use ISO format:

`YYYY-MM-DD`

## File

Use the exact file name.

Recommended patterns:

`YYYY-MM-DD_achievement_topic.md`

`YYYY-MM-DD_feedback_topic.md`

`YYYY-MM-DD_promotion-proof-point_topic.md`

`YYYY-MM-DD_portfolio-example_topic.md`

If agent-generated, follow:

`07_ai_agents/agent_file_naming.md`

## Evidence Type

Use one of the standard evidence types.

## Capability

Use the strongest capability category demonstrated by the evidence.

If multiple capabilities apply, list the primary one first.

## Status

Use one of the standard status values.

## Notes

Keep notes short.

Useful notes may include:

- useful for promotion review
- possible interview story
- supports business design positioning
- demonstrates systems thinking
- candidate portfolio example
- converted to resume bullet
- supports strategic opportunity

## Review Rhythm

Review this index during:

- monthly career evidence review
- monthly strategic opportunities review
- quarterly Strategic OS QA assessment
- promotion readiness review
- role application preparation

Do not review it more often than it creates value.

## Reuse Opportunities

Career evidence may later become:

- resume achievement bullet
- portfolio case study
- interview story
- promotion proof point
- development conversation example
- role application response
- LinkedIn or thought leadership seed
- strategic opportunity note
- capability narrative
- reputation positioning statement

## Privacy Guidance

Career evidence should be safe, truthful and reusable.

Do not index or store:

- raw confidential feedback
- private performance information about others
- sensitive stakeholder commentary
- confidential organisational extracts
- customer-level information
- private employee matters
- governed datasets
- internal documents copied as evidence
- claims that cannot be supported

Use privacy-safe summaries.

Instead of:

`Jane said I saved the channel strategy project after Team X failed`

Use:

`feedback indicates strong contribution to reframing a complex channel strategy challenge`

Instead of:

`complaint case 12345 proves I improved vulnerable customer handling`

Use:

`service review work demonstrated ability to connect customer vulnerability considerations to operational design`

## Quality Check Before Indexing

Before adding career evidence to the index, check:

- Is this evidence worth finding again?
- Is the file name clear?
- Is the topic privacy-safe?
- Is the evidence type accurate?
- Is the capability clear?
- Is the status accurate?
- Is the evidence specific?
- Is the outcome or value visible?
- Is sensitive material removed or abstracted?
- Could this support a future opportunity, promotion review or role application?

## Success Measures

This index is working when:

- useful career evidence is easy to find
- achievements are captured close to the work
- feedback is safely retained
- promotion readiness is easier to assess
- role applications are easier to prepare
- portfolio examples are easier to identify
- capability gaps become visible
- strategic opportunities connect to credible evidence

## Failure Modes

This index is failing if:

- every minor reflection is indexed
- evidence describes activity but not impact
- sensitive details appear in topics or notes
- claims are unsupported
- feedback is stored unsafely
- statuses are not updated
- evidence is captured too late
- the index becomes harder to maintain than useful

## Minimal Use Rule

Index only evidence that may help future positioning, progression or opportunity development.

A useful index is better than a complete index.
