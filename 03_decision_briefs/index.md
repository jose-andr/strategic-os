# Decision Briefs Index

## Purpose

This index tracks decision briefs stored in `03_decision_briefs/`.

Use it to make important decision-support outputs easier to find, compare, review and reuse.

The index should help show how Strategic OS supports the North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

This is not a complete decision register.

It is a lightweight navigation aid for decision briefs that may be useful again.

## What To Index

Index decision briefs that:

- support a meaningful decision
- clarify options or trade-offs
- contain reusable decision logic
- connect to a strategic opportunity
- affect project direction
- include stakeholder implications
- may become career evidence
- may be useful for future reviews

Do not index:

- rough notes
- temporary drafts
- low-value outputs
- sensitive material that should not be retained
- decisions that belong only in official organisational systems

## Status Values

Use simple status values.

| Status | Meaning |
| --- | --- |
| Draft | Still being shaped |
| Ready for review | Structured enough for human review |
| Used | Used to support a decision or conversation |
| Superseded | Replaced by a later brief or decision |
| Archived | Retained for reference only |
| Reusable | Contains decision logic worth reusing |

## Field Guidance

## Date

Use the date the brief was created or last meaningfully updated.

Use ISO format:

`YYYY-MM-DD`

## File

Use the exact file name.

Recommended pattern:

`YYYY-MM-DD_decision-brief_topic.md`

If agent-generated, follow:

`07_ai_agents/agent_file_naming.md`

## Type

Usually use:

`Decision Brief`

Other possible values:

- Problem Framing Note
- Options Assessment
- Trade-Off Summary
- Recommendation Note
- Decision Review

## Topic

Use a short, privacy-safe topic label.

Good examples:

- channel-mix-priorities
- operating-cadence
- stakeholder-alignment-risk
- service-account-metric-source
- project-scope-trade-off

Avoid:

- vague labels
- confidential stakeholder names
- customer case identifiers
- sensitive internal issue names

## Status

Use one of the standard status values.

## Notes

Keep notes short.

Useful notes may include:

- supports project direction
- reusable trade-off logic
- linked to opportunity review
- used in stakeholder discussion
- superseded by later recommendation
- candidate for lesson learned

## Review Rhythm

Review this index during:

- weekly inbox triage, where relevant
- monthly strategic opportunities review
- quarterly Strategic OS QA assessment
- project closure review

Do not review it more often than it creates value.

## Reuse Opportunities

A decision brief may later become:

- a project decision record
- a lesson learned
- a reusable framework
- a career evidence point
- a thought leadership seed
- a stakeholder alignment pattern
- a strategic opportunity
- a quarterly QA evidence item

## Privacy Guidance

Decision briefs often sit close to organisational context.

Do not index or store:

- customer-level records
- confidential source extracts
- private stakeholder commentary
- governed datasets
- sensitive employee matters
- raw operational data
- material that belongs only in official systems

Use privacy-safe labels and summaries.

Instead of:

`customer-case-12345-complaint-failure.md`

Use:

`complaint-handling-continuity-risk.md`

Instead of:

`jane-smith-blocking-project.md`

Use:

`senior-sponsor-alignment-risk.md`

## Quality Check Before Indexing

Before adding a decision brief to the index, check:

- Is the brief worth finding again?
- Is the file name clear?
- Is the topic privacy-safe?
- Is the status accurate?
- Is the decision logic reusable?
- Are evidence and assumptions separated?
- Is sensitive information removed or abstracted?
- Is the brief stored in the right folder?

## Success Measures

This index is working when:

- important decision briefs are easy to find
- decision logic can be reused
- superseded briefs are clearly marked
- quarterly review is easier
- project decisions are easier to trace
- strategic opportunities connect to decision evidence
- privacy-safe naming is maintained

## Failure Modes

This index is failing if:

- every minor note is indexed
- statuses are not updated
- sensitive details appear in topics or notes
- file names do not match the index
- outdated briefs appear current
- the index becomes harder to maintain than useful
- decision briefs are stored but never reviewed

## Minimal Use Rule

Index only decision briefs that may matter later.

A useful index is better than a complete index.
