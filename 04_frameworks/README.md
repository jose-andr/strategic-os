# Frameworks

## Purpose

This folder contains reusable Strategic OS frameworks, models, methods and operating patterns.

Use this folder to store structured ways of thinking that can be reused across decisions, projects, stakeholder work, career development and AI-enabled workflows.

Frameworks should help turn complexity into clearer judgement, better decisions and more strategic opportunities.

## What belongs here

- Decision frameworks
- Stakeholder frameworks
- Service design frameworks
- System mapping frameworks
- Analytics interpretation frameworks
- Opportunity assessment frameworks
- Career positioning frameworks
- QA and performance frameworks
- Operating models
- Workflow patterns
- Cross-tool architecture notes
- Privacy, governance and retention rules
- Strategic OS system maps

## What does not belong here

- Raw notes
- One-off project artefacts
- Completed decision briefs
- Full project documentation
- Personal career evidence
- Sensitive organisational source data
- Customer-level records
- Governed datasets
- Confidential extracts

Use this folder for reusable models, not project-specific records.

## Framework categories

Use these categories to organise frameworks over time:

### Decision frameworks

Reusable models for clarifying decisions, options, trade-offs, recommendations and next actions.

### Stakeholder frameworks

Reusable models for understanding alignment, influence, resistance, engagement sequence and communication needs.

### Service design frameworks

Reusable models for customer journeys, service blueprints, operating models, value flows and experience patterns.

### Systems thinking frameworks

Reusable models for mapping relationships, feedback loops, constraints, dependencies and system behaviour.

### Analytics interpretation frameworks

Reusable models for interpreting data, caveats, metric quality, contextual evidence and governed reporting.

### Opportunity frameworks

Reusable models for assessing strategic value, timing, stakeholder support, customer value, organisational value and trade-offs.

### Career positioning frameworks

Reusable models for translating work into evidence, achievements, promotion readiness and long-term positioning.

### QA and performance frameworks

Reusable models for assessing whether Strategic OS is working, useful, safe and aligned to the North Star.

## Suggested naming

Use clear, lowercase filenames with underscores.

Examples:

- `intake_to_output_workflow.md`
- `operating_cadence.md`
- `career_evidence_architecture.md`
- `strategic_os_qa_model.md`

## Review rhythm

Review this folder monthly or quarterly.

Use the review to ask:

- Which frameworks are being used?
- Which frameworks are too complex?
- Which frameworks should become templates?
- Which project-specific patterns should be generalised?
- Which frameworks support the North Star?
- Which frameworks should be archived or simplified?

## Review checklist

Before saving a framework, check:

- [ ] Is this reusable beyond one project?
- [ ] Does it help make better strategic decisions?
- [ ] Is the framework clear and practical?
- [ ] Is it different from an existing template?
- [ ] Is it free of sensitive organisational source data?
- [ ] Does it support Strategic OS principles?
- [ ] Is there a clear example or use case?
