# Operating Cadence

## Purpose

This framework defines the operating rhythm for Strategic OS.

Strategic OS is not only a repository for storing knowledge. It is a personal operating system for turning complexity, signals, evidence, relationships and work into clearer strategic decisions.

The operating cadence keeps the system alive.

It creates regular moments to:

* clarify priorities
* review incoming signals
* make decisions visible
* identify strategic opportunities
* maintain stakeholder alignment
* convert experience into reusable knowledge
* protect focus
* review agent and template usefulness
* improve the system over time

## Strategic OS North Star

> Increase strategic opportunities by helping leaders make better strategic decisions.

## Core Principle

Strategic OS should be reviewed at the rhythm of the work.

Daily cadence supports focus.

Weekly cadence supports sensemaking.

Monthly cadence supports opportunity development.

Quarterly cadence supports strategic alignment and system quality.

Use the smallest useful cadence. Do not turn Strategic OS into administration.

## Cadence Overview

| Rhythm    | Main Focus  | Primary Function                                                 | Typical Agent                            |
| --------- | ----------- | ---------------------------------------------------------------- | ---------------------------------------- |
| Daily     | Focus       | Clarify priorities, decisions and follow-ups                     | Chief of Staff Agent                     |
| Weekly    | Sensemaking | Convert activity into patterns, actions and alignment            | Chief of Staff Agent / Sensemaking Agent |
| Monthly   | Opportunity | Develop strategic opportunities, evidence and reusable knowledge | Career Architect / Sensemaking Agent     |
| Quarterly | Quality     | Review system health, agent performance and North Star alignment | Sensemaking Agent / Chief of Staff Agent |

## Daily Cadence

## Purpose

The daily cadence helps translate current complexity into immediate focus.

It should be lightweight, practical and action-oriented.

Use it to decide what matters today, what needs attention and what should not be lost.

## Key Questions

Ask:

* What matters most today?
* What decisions need attention?
* What conversations or stakeholders need preparation?
* What follow-ups are at risk of slipping?
* What admin or operational task could create friction if ignored?
* What can be deferred, batched or dropped?
* What should be captured before it is lost?
* What needs human judgement before action?

## Recommended Activities

### Chief of Staff Daily Briefing

Use the Chief of Staff Agent to create a short daily briefing.

Recommended template:

`10_templates/daily_briefing.md`

The briefing should identify:

* top priorities
* key meetings
* decisions required
* stakeholder sensitivities
* follow-ups
* risks or blockers
* useful preparation notes
* focus protection actions
* escalation points

Most daily briefings should remain temporary working notes.

Store only if the briefing creates durable value, decision context, project continuity, stakeholder evidence, career evidence or a reusable lesson.

### Calendar And Priority Check

Review the day’s calendar against current strategic priorities.

Look for:

* meetings that need preparation
* meetings that should be reframed
* opportunities to influence a decision
* commitments that need follow-up
* work that should be deferred
* capacity risks
* conflicts between urgent work and strategic work

The Chief of Staff Agent may recommend planning changes, but it must not change calendar events, contact stakeholders or alter commitments.

### Admin Task Check

Review small operational tasks that could create friction if ignored.

Recommended template:

`10_templates/admin_task_list.md`

Use this only when small tasks need grouping.

Examples:

* emails requiring response
* meeting notes to send
* documents to update
* actions promised to others
* tasks blocking another person
* repository hygiene tasks
* low-risk coordination actions

Admin work should not crowd out strategic work.

If an admin task hides a decision, convert it into a decision brief.

### Follow-Up Check

Review promised follow-ups and dependencies.

Recommended template:

`10_templates/follow_up_tracker.md`

Use this when there are open loops that should not rely on memory.

Track:

* promised follow-ups
* follow-ups waiting on others
* blocked follow-ups
* stakeholder-sensitive follow-ups
* decision-linked follow-ups

Escalate any follow-up that implies commitment, affects stakeholder expectations or requires formal approval.

### Decision-Required Check

Identify whether anything today needs a clear decision.

Capture:

* the decision required
* the options
* what evidence is available
* what is uncertain
* who needs to be involved
* the next action
* whether a decision brief is needed

Use:

`10_templates/decision_brief.md`

when the decision needs structure, recommendation or escalation.

## Daily Outputs

Daily outputs may include:

* daily briefing
* priority list
* decision-required list
* meeting preparation note
* follow-up tracker
* admin task list
* inbox item
* project note
* stakeholder preparation note

## Daily Storage Guidance

Store only material that has reusable value.

Use:

* `00_inbox/` for unsorted notes or signals
* `03_decision_briefs/` for decision-related material
* `06_stakeholder_patterns/` for reusable stakeholder observations
* `08_projects/` for project-specific operating notes
* `01_career/` for career evidence
* `05_lessons_learned/` for reusable learning

Do not store:

* temporary task noise
* routine daily briefings with no reuse value
* sensitive organisational material
* raw confidential information
* customer-level records
* unreviewed stakeholder commentary

## Weekly Cadence

## Purpose

The weekly cadence converts activity into sensemaking.

It helps identify what changed, what matters, what is stuck and what should move forward.

## Key Questions

Ask:

* What signals emerged this week?
* What patterns are becoming visible?
* What decisions moved forward?
* What decisions remain unclear?
* Where is stakeholder alignment improving or deteriorating?
* What should be shipped, socialised, refined or stopped?
* What opportunities emerged?
* What knowledge should be saved?
* What should be discarded?
* What needs to be carried into next week?

## Recommended Activities

### Weekly Operating Review

Use the Chief of Staff Agent to review the week.

Recommended template:

`10_templates/weekly_operating_review.md`

The review should identify:

* what moved forward
* what stalled
* decisions made
* decisions still required
* follow-ups closed or carried forward
* stakeholder movement
* work to ship, socialise, refine or stop
* risks and blockers
* strategic opportunity signals
* lessons learned
* career evidence
* next week’s focus

The weekly review is the main bridge between daily activity and durable Strategic OS knowledge.

### Sensemaking Review

Review notes, meetings, conversations and project signals from the week.

Look for:

* recurring themes
* unresolved tensions
* weak signals
* emerging risks
* stakeholder concerns
* decision bottlenecks
* opportunities for reframing
* evidence that supports or challenges existing assumptions

Useful outputs:

* sensemaking note
* problem framing
* pattern summary
* issue diagnosis
* opportunity statement
* decision brief input

Recommended agent:

Sensemaking Agent

### Shipping Review

Review what moved from thinking into action.

Ask:

* What was shipped?
* What is still too abstract?
* What needs a smaller next step?
* What is blocked?
* What can be simplified?
* What should be stopped?
* What should be socialised before further refinement?

Useful outputs:

* shipping recommendation
* next-action list
* project update
* delivery risk note
* lesson learned

Recommended agent:

Shipping Coach

Recommended template:

`10_templates/shipping_recommendation.md`

### Stakeholder Alignment Check

Review key stakeholder relationships and alignment conditions.

Ask:

* Who is aligned?
* Who is uncertain?
* Who has not been heard?
* Who may be affected by a decision?
* What concerns need to be surfaced?
* What language or framing may help?
* What assumptions need validation?

Useful outputs:

* stakeholder alignment assessment
* stakeholder pattern note
* engagement sequence
* influence map
* meeting preparation note

Recommended agent:

Stakeholder Journey Agent

Recommended template:

`10_templates/stakeholder_alignment_assessment.md`

### Inbox Triage

Review `00_inbox/` and decide what to do with captured material.

Each item should be:

* moved
* developed
* merged
* archived
* deleted
* kept temporarily with a review date

Suggested destinations:

* `01_career/`
* `02_strategic_opportunities/`
* `03_decision_briefs/`
* `04_frameworks/`
* `05_lessons_learned/`
* `06_stakeholder_patterns/`
* `08_projects/`
* `09_thought_leadership/`
* `10_templates/`
* `11_analytics/`

## Monthly Cadence

* What strategic opportunities are emerging?
* What evidence has been created?
* What work strengthened the desired reputation?
* What patterns are worth formalising?
* What templates or frameworks should be improved?
* What should be made more visible?
* What should be retired?
* What analytical caveats or source rules need updating?
* What repeated issue should become a decision, lesson or framework?

`02_strategic_opportunities/`

* opportunity review
* opportunity pipeline update
* next-step recommendation

* decision brief

`10_templates/opportunity_review.md`

* what should be avoided

`10_templates/lesson_learned.md`

## Quarterly Cadence

## Purpose

The quarterly cadence reviews whether Strategic OS is still aligned to its North Star.

It is a deeper system review focused on quality, direction, usefulness, safety and strategic outcomes.

## Key Questions

Ask:

* Is Strategic OS helping create better strategic decisions?
* Is it increasing strategic opportunities?
* Is the repository still clean and useful?
* Are the agents performing useful roles?
* Are the frameworks still relevant?
* Are templates practical?
* Is the system producing evidence of value?
* Are privacy and governance boundaries respected?
* Are analytical caveats and source rules maintained?
* What should change next quarter?

## Recommended Activities

### Strategic OS QA Assessment

Review the system as a whole.

Recommended template:

`10_templates/quarterly_strategic_os_assessment.md`

Assess:

* repository clarity
* folder hygiene
* duplication
* outdated material
* missing files
* confusing structures
* whether files remain durable and reusable
* whether sensitive information is excluded
* whether outputs support the North Star

Useful outputs:

* QA assessment
* clean-up list
* repository improvement backlog
* framework update
* template update

### Agent Performance Review

Review AI agent usefulness.

Assess each active agent against:

* clarity of role
* quality of outputs
* usefulness to decisions
* failure modes
* human-in-the-loop safeguards
* autonomy boundaries
* escalation rules
* whether the agent should continue, change or be retired

Useful outputs:

* agent review note
* agent spec update
* prompt improvement
* autonomy rule update
* escalation rule update
* retired agent note

Review:

* Sensemaking Agent
* Stakeholder Journey Agent
* Shipping Coach
* Career Architect
* Chief of Staff Agent
* Domain Analytical Agents

### Template Usefulness Review

Review templates for actual use.

Ask:

* Which templates saved time?
* Which templates created friction?
* Which templates were too long?
* Which templates were not used?
* Which templates should be simplified?
* Which outputs need a template that does not yet exist?
* Which templates encouraged unnecessary storage?

Useful outputs:

* template improvement backlog
* template simplification
* template merge or retirement
* new template requirement

### Career Evidence Review

Review accumulated evidence of strategic value.

Look for:

* decisions influenced
* systems clarified
* stakeholders aligned
* projects improved
* risks reduced
* opportunities created
* reusable frameworks developed
* AI-enabled capability demonstrated
* customer or public value supported

Useful outputs:

* career evidence register
* achievement narrative
* portfolio example
* promotion or role-readiness evidence
* development priority

### Repository Hygiene Review

Review repository structure and content quality.

Check:

* file names
* folder fit
* broken or duplicated concepts
* outdated notes
* sensitive content risk
* files that should be archived
* files that should be promoted into templates or frameworks
* temporary Chief of Staff outputs that should be deleted
* analytical files stored in the correct domain folder

Useful outputs:

* cleaned repository
* archived files
* improved naming
* consolidated frameworks
* updated README files only when explicitly requested as an end-of-day completion pass

### North Star Alignment Review

Assess whether the system is still aligned to the Strategic OS North Star.

Ask:

* Did the system help create better strategic decisions?
* Did it improve stakeholder alignment?
* Did it increase strategic opportunities?
* Did it help turn complexity into useful structure?
* Did it strengthen the desired reputation?
* What should be emphasised next quarter?
* What should be stopped because it does not support the North Star?

Useful outputs:

* quarterly reflection
* strategic direction note
* next-quarter priorities
* system improvement decision

## Storage Discipline

Most daily material should not be stored.

Weekly material should be stored only when it creates durable value.

Monthly material should usually become structured knowledge.

Quarterly material should be retained as system evidence unless it contains sensitive information.

Store:

* decisions
* strategic opportunities
* lessons learned
* career evidence
* stakeholder patterns
* project continuity
* reusable frameworks
* reusable templates
* reviewed analytical caveats and source rules

Do not store:

* routine daily notes
* completed admin lists
* temporary reminders
* raw organisational source data
* customer-level records
* governed datasets
* confidential extracts
* sensitive stakeholder commentary
* private HR material
* unreviewed assumptions about people

## Success Measures

The operating cadence is working when:

* daily focus is clearer
* follow-ups are less likely to be missed
* weekly patterns become visible
* decisions are easier to frame
* stakeholder alignment improves
* strategic opportunities are captured and reviewed
* career evidence accumulates over time
* lessons are reused
* templates and agents are improved through use
* analytical caveats remain visible
* the repository stays clean
* the system supports the North Star

## Failure Modes

The operating cadence is failing if:

* daily reviews become too heavy
* weekly reviews become diary entries
* monthly reviews are skipped
* quarterly reviews are performative
* the inbox becomes permanent storage
* temporary Chief of Staff outputs create clutter
* opportunities are captured but never reviewed
* career evidence is assembled only at the last minute
* agents are used without review
* templates create more friction than value
* analytical outputs are stored without caveats
* privacy and governance checks are skipped
* the cadence becomes administration rather than leverage

## Minimal Use Rule

Use only the cadence that creates value.

A five-minute review that improves judgement is better than a complete ritual that creates clutter.

The cadence should make the Strategic OS easier to use, not harder to maintain.
