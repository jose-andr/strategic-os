# Lessons Learned Index

## Purpose

This index tracks reusable lessons stored in `05_lessons_learned/`.

Use it to make lessons easier to find, review, apply and convert into frameworks, templates, project patterns or career evidence.

The index supports the Strategic OS North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

This is not a complete diary of everything learned.

It is a lightweight navigation aid for lessons that may be useful again.

## What To Index

Index lessons that:

- reveal a reusable pattern
- improve future decision-making
- clarify a failure mode
- capture a useful project insight
- strengthen stakeholder practice
- improve shipping discipline
- support career evidence
- could become a framework or template
- may help future projects move faster or with less risk

Do not index:

- temporary reflections
- low-value observations
- sensitive material that should not be retained
- raw project notes
- private stakeholder commentary
- lessons that are too vague to reuse

## Status Values

Use simple status values.

| Status | Meaning |
| --- | --- |
| Draft | Lesson is still being shaped |
| Active | Lesson is current and useful |
| Reusable | Lesson is strong enough to apply elsewhere |
| Converted | Lesson has become a framework, template, project pattern or career evidence |
| Superseded | Replaced by a stronger or more current lesson |
| Archived | Retained for history only |

## Field Guidance

## Date

Use the date the lesson was created or last meaningfully updated.

Use ISO format:

`YYYY-MM-DD`

## File

Use the exact file name.

Recommended pattern:

`YYYY-MM-DD_lesson-learned_topic.md`

If agent-generated, follow:

`07_ai_agents/agent_file_naming.md`

## Type

Usually use:

`Lesson Learned`

Other possible values:

- Project Lesson
- Stakeholder Lesson
- Shipping Lesson
- Decision Lesson
- Career Lesson
- Analytics Lesson
- Framework Lesson

## Topic

Use a short, privacy-safe topic label.

Good examples:

- channel-strategy-framing
- stakeholder-alignment
- decision-quality
- operating-cadence
- analytics-caveats
- project-folder-setup
- shipping-before-perfect

Avoid:

- vague labels
- confidential stakeholder names
- customer case identifiers
- sensitive internal issue names

## Status

Use one of the standard status values.

## Notes

Keep notes short.

Useful notes may include:

- candidate for framework
- useful project pattern
- supports career evidence
- reusable facilitation principle
- converted to template
- superseded by later project lesson

## Review Rhythm

Review this index during:

- monthly lessons learned review
- project closure review
- quarterly Strategic OS QA assessment
- career evidence review, where relevant

Do not review it more often than it creates value.

## Reuse Opportunities

A lesson may later become:

- a reusable framework
- a template
- a project folder pattern
- a career evidence point
- a promotion proof point
- a thought leadership seed
- a stakeholder pattern
- a decision principle
- an agent prompt improvement

## Privacy Guidance

Lessons should preserve learning without exposing sensitive detail.

Do not index or store:

- customer-level records
- confidential source extracts
- private stakeholder commentary
- governed datasets
- sensitive employee matters
- raw operational data
- material that belongs only in official systems

Use privacy-safe labels and summaries.

Instead of:

`jane-smith-blocked-workshop.md`

Use:

`senior-stakeholder-readiness-risk.md`

Instead of:

`customer-case-12345-status-failure.md`

Use:

`status-visibility-customer-effort-pattern.md`

## Quality Check Before Indexing

Before adding a lesson to the index, check:

- Is this lesson worth finding again?
- Is the file name clear?
- Is the topic privacy-safe?
- Is the status accurate?
- Is the lesson reusable?
- Is the evidence separated from interpretation?
- Is sensitive information removed or abstracted?
- Could this become a framework, template or career proof point?

## Success Measures

This index is working when:

- useful lessons are easy to find
- project learning is reused
- repeated mistakes become visible
- strong lessons become frameworks or templates
- career evidence is easier to assemble
- quarterly review is easier
- privacy-safe naming is maintained

## Failure Modes

This index is failing if:

- every minor reflection is indexed
- statuses are not updated
- sensitive details appear in topics or notes
- file names do not match the index
- vague lessons clutter the folder
- lessons are captured but never reused
- the index becomes harder to maintain than useful

## Minimal Use Rule

Index only lessons that may change future behaviour.

A useful index is better than a complete index.
