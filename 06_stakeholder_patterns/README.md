# Stakeholder Patterns

## Purpose

This folder contains reusable stakeholder patterns, influence dynamics, engagement approaches and alignment lessons.

Use this folder to capture patterns that help bring people along the journey and improve stakeholder alignment over time.

Stakeholder patterns should support better strategic decisions by making influence, concerns, motivations and engagement needs clearer.

## What belongs here

- Stakeholder alignment assessments
- Reusable influence patterns
- Engagement patterns
- Communication framing patterns
- Stakeholder journey notes
- Lessons about resistance or support
- Patterns from workshops, meetings or decision processes
- Safe summaries of stakeholder needs or concerns
- Reusable approaches for bringing people along

## What does not belong here

- Sensitive stakeholder commentary
- Personal judgements about individuals
- Confidential meeting records
- Raw organisational source data
- Customer-level records
- Governed datasets
- Private organisational documents
- Full project documentation

Use careful, fair and generalised summaries instead.

## Pattern types

Use these categories where useful:

| Pattern type | Use for |
|---|---|
| Alignment pattern | How support, uncertainty or resistance formed |
| Influence pattern | How formal or informal influence shaped progress |
| Engagement pattern | What helped stakeholders understand, support or act |
| Communication pattern | What framing worked for different audiences |
| Decision pattern | What helped stakeholders move toward a decision |
| Resistance pattern | What concerns, risks or blockers needed attention |
| Trust pattern | What built or weakened confidence |

## Suggested naming

Use date-first filenames.

Examples:

- `2026-06-15_stakeholder-alignment_channel-strategy.md`
- `2026-06-15_stakeholder-pattern_executive-briefing.md`
- `2026-06-15_stakeholder-pattern_cross-functional-alignment.md`

Avoid using personal stakeholder names in filenames.

## Review rhythm

Review this folder monthly or before major stakeholder conversations.

Use the review to ask:

- What alignment patterns are repeating?
- What concerns need earlier attention?
- What communication framing is working?
- What influence pathways are becoming clearer?
- What should become a reusable framework?
- What should inform future stakeholder engagement?

## Review checklist

Before saving a stakeholder pattern, check:

- [ ] Is the pattern fair and evidence-aware?
- [ ] Are assumptions separated from observations?
- [ ] Is sensitive information removed or generalised?
- [ ] Does the pattern avoid personal judgement?
- [ ] Is the reuse value clear?
- [ ] Could this improve a future stakeholder conversation?
- [ ] Is the file safe to retain in Strategic OS?
