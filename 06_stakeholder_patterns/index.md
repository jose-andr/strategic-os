# Stakeholder Patterns Index

## Purpose

This index tracks reusable stakeholder patterns stored in `06_stakeholder_patterns/`.

Use it to make stakeholder alignment, influence, communication and engagement lessons easier to find and apply.

The index supports the Strategic OS North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

This is not a stakeholder register.

It is not a record of private comments.

It is a lightweight index of reusable, privacy-safe patterns.

## What To Index

Index stakeholder patterns that:

- reveal an alignment condition
- improve future stakeholder preparation
- clarify influence dynamics
- identify a communication pattern
- support better decision framing
- show a recurring source of resistance or concern
- help build trust
- support project or opportunity development
- may be useful across multiple contexts

Do not index:

- private stakeholder commentary
- named criticism
- confidential relationship dynamics
- sensitive employee matters
- raw meeting notes
- unverified motives
- low-value observations
- material that would breach trust if read out of context

## Status Values

Use simple status values.

| Status | Meaning |
| --- | --- |
| Draft | Pattern is still being shaped |
| Active | Pattern is current and useful |
| Reusable | Pattern is strong enough to apply elsewhere |
| Converted | Pattern has become a framework, template, lesson or project note |
| Superseded | Replaced by a stronger or more current pattern |
| Archived | Retained for history only |

## Pattern Types

Use one of these pattern types where possible.

| Pattern Type | Description |
| --- | --- |
| Alignment Pattern | Conditions that help or block shared understanding |
| Influence Pattern | How support, trust or sponsorship develops |
| Communication Pattern | Framing, language or channel considerations |
| Resistance Pattern | Recurring concerns, objections or sources of hesitation |
| Decision Readiness Pattern | Conditions needed before stakeholders can decide |
| Engagement Pattern | How to involve people effectively |
| Trust Pattern | What strengthens or weakens confidence |
| Sponsorship Pattern | How senior support is created or maintained |
| Cross-Team Pattern | How groups interact, hand over or coordinate |
| Sensitive Context Pattern | Privacy-safe abstraction of a delicate stakeholder dynamic |

## Field Guidance

## Date

Use the date the pattern was created or last meaningfully updated.

Use ISO format:

`YYYY-MM-DD`

## File

Use the exact file name.

Recommended pattern:

`YYYY-MM-DD_stakeholder-pattern_topic.md`

If agent-generated, follow:

`07_ai_agents/agent_file_naming.md`

## Pattern Type

Use one of the standard pattern types.

## Topic

Use a short, privacy-safe topic label.

Good examples:

- senior-sponsor-readiness
- cross-team-alignment
- decision-owner-clarity
- workshop-framing
- operating-model-concerns
- evidence-before-recommendation

Avoid:

- named stakeholders
- team blame
- confidential project names
- private relationship details
- customer identifiers
- sensitive internal issue names

## Status

Use one of the standard status values.

## Notes

Keep notes short.

Useful notes may include:

- useful for sponsor preparation
- reusable framing principle
- supports decision-readiness work
- candidate for stakeholder template
- converted to lesson learned
- relevant to project setup

## Review Rhythm

Review this index during:

- monthly stakeholder alignment review
- project closure review
- strategic opportunity review, where relevant
- quarterly Strategic OS QA assessment

Do not review it more often than it creates value.

## Reuse Opportunities

A stakeholder pattern may later become:

- a stakeholder alignment assessment
- a project engagement note
- a lesson learned
- a reusable communication frame
- a workshop design principle
- a decision brief caveat
- a career evidence point
- a thought leadership seed
- an agent prompt improvement

## Privacy Guidance

Stakeholder patterns should protect trust.

Do not index or store:

- private stakeholder comments
- named criticism
- sensitive relationship dynamics
- unverified motives
- confidential organisational politics
- private employee matters
- customer-level information
- raw meeting notes

Use privacy-safe abstractions.

Instead of:

`Jane is blocking the channel strategy because she disagrees with Mark`

Use:

`cross-functional sponsorship alignment risk`

Instead of:

`Team X does not trust the data team`

Use:

`analytics confidence and source-trust pattern`

Instead of:

`Sponsor was annoyed in the meeting`

Use:

`senior stakeholder expectation-setting pattern`

## Quality Check Before Indexing

Before adding a stakeholder pattern to the index, check:

- Is this pattern worth finding again?
- Is the file name clear?
- Is the topic privacy-safe?
- Is the status accurate?
- Is the pattern reusable?
- Are assumptions separated from evidence?
- Is sensitive commentary removed or abstracted?
- Would this still feel safe if reviewed later?
- Could this support a future decision, project or opportunity?

## Success Measures

This index is working when:

- reusable stakeholder patterns are easy to find
- stakeholder preparation improves
- alignment risks are surfaced earlier
- sensitive details are protected
- project engagement becomes more deliberate
- decision framing improves
- patterns can be reused across contexts
- quarterly review is easier

## Failure Modes

This index is failing if:

- it becomes a stakeholder register
- private commentary is stored
- named criticism appears in topics or notes
- unverified motives are treated as facts
- every minor observation is indexed
- statuses are not updated
- patterns are captured but not reused
- the index becomes harder to maintain than useful

## Minimal Use Rule

Index only stakeholder patterns that may improve future alignment, trust or decision quality.

A useful index is better than a complete index.
