# Agent Output Index Pattern

## Purpose

This file defines a lightweight index pattern for saved Strategic OS agent outputs.

The index helps make agent-generated material easier to find, review, reuse and retire.

It is not intended to become a heavy register or administrative burden.

## Core Principle

Only index outputs that are worth finding again.

The goal is to preserve useful thinking, not catalogue every interaction with an agent.

## When To Use An Output Index

Use an output index when a folder contains multiple saved agent outputs that may need to be reviewed over time.

Good candidates:

- `01_career/`
- `02_strategic_opportunities/`
- `03_decision_briefs/`
- `05_lessons_learned/`
- `06_stakeholder_patterns/`
- `08_projects/`
- `09_thought_leadership/`

An index is especially useful when outputs relate to:

- decisions
- opportunities
- stakeholder alignment
- career evidence
- lessons learned
- reusable frameworks
- recurring projects
- quarterly reviews

## When Not To Use An Output Index

Do not create an index for:

- temporary notes
- low-value drafts
- daily briefings that are not reused
- sensitive material that should not be stored
- one-off outputs with no future value
- folders with only one or two stable files

## Recommended Index File Name

Use:

`agent_output_index.md`

Place it inside the folder it indexes.

Example:

`03_decision_briefs/agent_output_index.md`

## Standard Index Structure

Use this structure:

# Agent Output Index

## Purpose

This index tracks saved agent outputs in this folder that may be useful for review, reuse or future decision-making.

## Index

| Date | Agent | Output Type | Topic | File | Status | Notes |
| --- | --- | --- | --- | --- | --- | --- |
| YYYY-MM-DD | Agent Name | Output Type | Topic | file-name.md | Active | Short note |

## Status Values

Use simple status values:

- Active
- Draft
- Superseded
- Archived
- Reusable
- Review

## Review Notes

Use this section for lightweight observations about patterns emerging across saved outputs.

## Maintenance

Review this index during the monthly or quarterly Strategic OS cadence.

## Field Definitions

### Date

The date the output was created or saved.

Use ISO format:

`YYYY-MM-DD`

### Agent

The agent that produced or supported the output.

Examples:

- Sensemaking Agent
- Stakeholder Journey Agent
- Shipping Coach
- Career Architect
- Chief of Staff Agent

### Output Type

The kind of output saved.

Examples:

- Daily Briefing
- Decision Brief
- Stakeholder Alignment Assessment
- Shipping Recommendation
- Opportunity Review
- Career Guidance
- Promotion Readiness Assessment
- Sensemaking Summary
- Lesson Learned

### Topic

A short description of the subject.

Examples:

- channel strategy
- stakeholder alignment
- operating cadence
- service account review
- business design positioning
- decision quality

### File

The file name of the saved output.

Use the naming conventions in:

`07_ai_agents/agent_file_naming.md`

### Status

The current usefulness of the output.

Use:

- Active: currently relevant
- Draft: unfinished or provisional
- Superseded: replaced by a later output
- Archived: retained for history only
- Reusable: useful as a pattern or reference
- Review: needs human review before reuse

### Notes

Use brief notes only.

Good notes:

- Supports quarterly review
- Useful decision logic
- Reusable stakeholder framing
- Superseded by later brief
- Candidate for template

Avoid long explanations.

## Review Cadence

Review indexes during:

- weekly inbox triage
- monthly opportunity review
- quarterly repository hygiene review

Do not review every index every week.

Use judgement.

## Privacy Guidance

Indexes should not expose sensitive context.

Do not include:

- customer names
- confidential stakeholder names
- raw organisational source references
- sensitive internal issues
- private employee matters
- governed dataset details

Use abstract, reusable labels.

Instead of:

`Jane Smith conflict`

Use:

`senior sponsor alignment risk`

Instead of:

`customer complaint case 12345`

Use:

`complaint handling escalation pattern`

## Success Measures

The index pattern is working when:

- useful outputs are easy to find
- related outputs can be compared
- reusable thinking is visible
- old drafts do not clutter the system
- quarterly reviews are easier
- career evidence is easier to assemble
- decision history is easier to trace
- sensitive details are not exposed

## Failure Modes

The index pattern is failing if:

- every minor output is indexed
- indexes become more work than value
- sensitive details appear in index rows
- status values are ignored
- outdated outputs are treated as current
- the index is not reviewed
- file names and index entries do not match

## Maintenance

During quarterly repository hygiene review:

- remove low-value entries
- mark superseded outputs
- archive outdated material
- update missing links or file names
- identify reusable patterns
- promote valuable outputs into frameworks or templates

## Minimal Use Rule

When in doubt, keep the index light.

A useful index is better than a complete index.

The index should help Strategic OS remain usable, not turn it into administration.
