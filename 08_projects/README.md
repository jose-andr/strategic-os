# Projects

## Purpose

This folder contains project-specific Strategic OS artefacts.

Use this folder for work that belongs to a specific project, initiative, service, domain or delivery effort and has not yet become a reusable framework, template, lesson or career evidence.

Project folders help keep context, decisions, outputs and lessons organised without mixing project-specific detail into the wider Strategic OS framework.

## What belongs here

- Project context
- Project-specific decision notes
- Project artefacts
- Project outputs
- Stakeholder notes
- Delivery notes
- Project-specific lessons
- Working drafts related to a project
- Safe summaries of project evidence
- Links or references to organisational systems of record

## What does not belong here

- Raw organisational source data
- Customer-level records
- Governed datasets
- Confidential extracts
- Sensitive stakeholder commentary
- General reusable frameworks
- Final templates
- Career evidence that has already been converted into proof points

Use summaries, caveats, links and reusable learning instead.

## Recommended project structure

Use this pattern for new projects:

    08_projects/
    └── project-name/
        ├── README.md
        ├── context.md
        ├── decisions.md
        ├── stakeholder-notes.md
        ├── outputs/
        ├── lessons-learned.md
        └── archive/

## File purposes

### `README.md`

Project overview, purpose, scope, key links and current status.

### `context.md`

Background, problem framing, project drivers, constraints and assumptions.

### `decisions.md`

Decision log, recommendations, trade-offs and open questions.

### `stakeholder-notes.md`

Safe, generalised notes about stakeholder alignment, engagement needs and communication considerations.

### `outputs/`

Project-specific artefacts, drafts, summaries or deliverables.

### `lessons-learned.md`

Reusable learning from the project that may later move to `05_lessons_learned/`.

### `archive/`

Superseded or inactive project material that is safe to retain.

## Suggested naming

Use lowercase project folder names with hyphens.

Examples:

- `08_projects/channel-strategy/`
- `08_projects/service-account/`
- `08_projects/strategic-os-framework/`

For files inside a project folder, use date-first filenames where helpful.

Examples:

- `2026-06-15_decision-brief_next-step.md`
- `2026-06-15_stakeholder-alignment_review.md`
- `2026-06-15_lesson-learned_delivery-pattern.md`

## Review rhythm

Review active project folders weekly or before major project conversations.

Use the review to ask:

- What decisions are open?
- What needs stakeholder alignment?
- What artefacts are ready to ship or socialise?
- What lessons are emerging?
- What should become reusable Strategic OS knowledge?
- What should become career evidence?
- What should be archived?

## Review checklist

Before saving project material, check:

- [ ] Is this project-specific rather than reusable?
- [ ] Is sensitive information removed or generalised?
- [ ] Are links used instead of copying source data?
- [ ] Is the current status clear?
- [ ] Are decisions and open questions visible?
- [ ] Should any part of this be moved to another Strategic OS folder?
- [ ] Is the material safe to retain?
