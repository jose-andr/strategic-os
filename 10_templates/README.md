# Templates

This folder contains reusable Strategic OS templates.

Templates support repeatable outputs for decision-making, learning, stakeholder alignment, shipping discipline and career development.

Templates should be practical, markdown-first and easy to reuse across projects, opportunities and agent workflows.

## Template principles

Templates should:

1. Support better strategic decisions.
2. Separate evidence, assumptions, interpretation and recommendation.
3. Be easy to complete in markdown.
4. Help move work forward.
5. Create reusable knowledge.
6. Avoid storing sensitive organisational source data.
7. Support human judgement rather than replace it.

## Template naming pattern

Use lowercase filenames with hyphens or underscores.

Preferred pattern:

`template_name.md`

Examples:

- `decision_brief.md`
- `stakeholder_alignment_assessment.md`
- `shipping_recommendation.md`

## Adding new templates

When adding a new template, include:

- Purpose
- When to use
- Main sections
- Final recommendation or output
- Review checklist

If the template supports an agent, reference the relevant agent in:
