# Admin Task List

## Purpose

This template helps the Chief of Staff Agent capture, sort and manage low-risk administrative tasks without letting them crowd out strategic work.

It is designed to help the user:

* see small operational tasks clearly
* group similar admin work
* identify what can be deferred
* avoid missed commitments
* separate admin from decisions
* protect attention for higher-value strategic work
* reduce operational drag

North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

## When To Use

Use this template when there are multiple small tasks, loose ends or operational follow-ups that need to be tracked but do not require a full decision brief or project artefact.

Use it for:

* simple admin follow-ups
* scheduling tasks
* document clean-up
* reminders
* low-risk coordination tasks
* routine updates
* small preparation actions
* inbox triage outputs
* tasks carried forward from a daily briefing
* tasks carried forward from a weekly operating review

Do not use this template for:

* formal decisions
* sensitive stakeholder communication
* governance actions
* customer-level records
* confidential organisational source data
* project plans that belong in `08_projects/`
* strategic opportunities that belong in `02_strategic_opportunities/`

* `00_inbox/` for unprocessed notes
* `03_decision_briefs/` for decision-required items
* `05_lessons_learned/` for reusable learning
* `06_stakeholder_patterns/` for reusable stakeholder insight
* `08_projects/` for project-specific admin
* `10_templates/` for reusable template improvements

Do not store:

* temporary task noise
* sensitive stakeholder commentary
* raw organisational source data
* customer-level records
* confidential extracts
* duplicated reminders

## Review Checklist

Before using the admin task list, check:

* Are tasks clearly described?
* Are admin tasks separated from decisions?
* Are due dates or timing expectations clear?
* Are owners or dependencies visible?
* Are similar tasks batched?
* Are low-value tasks deferred or dropped?
* Are stakeholder-sensitive tasks flagged?
* Are high-risk tasks escalated out of the admin list?
* Is strategic focus protected?
* Is the list short enough to use?
* Is sensitive information removed or generalised?
* Does the list reduce operational drag rather than create more work?

## Output Quality Bar

A useful admin task list should be:

* lightweight
* easy to scan
* grouped by action type
* clear about urgency
* realistic about capacity
* explicit about dependencies
* safe to use
* focused on reducing drag
* subordinate to strategic priorities

## Failure Modes

This template is not working if:

* it becomes a dumping ground
* every task is treated as urgent
* admin work crowds out strategic work
* decisions are hidden as tasks
* sensitive communication is treated as routine admin
* the list is too long to act on
* completed tasks are not cleared
* low-value tasks are repeatedly carried forward
* it creates more maintenance burden than clarity
