# Daily Briefing

## Purpose

This template helps the Chief of Staff Agent turn daily context into a clear operating brief.

It is designed to help the user:

* focus attention
* prepare for the day
* surface decisions
* reduce missed follow-ups
* protect capacity
* connect day-to-day work to the Strategic OS North Star

North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

## When To Use

Use this template when preparing for the day, especially when there are:

* multiple meetings
* competing priorities
* stakeholder conversations
* open decisions
* admin follow-ups
* delivery risks
* unclear next actions
* important context that should not be lost

This template can be used daily or only on high-complexity days.

## Inputs

Useful inputs may include:

* calendar notes
* meeting list
* current priorities
* project notes
* open tasks
* follow-up items
* stakeholder concerns
* decision notes
* inbox items
* deadlines
* unresolved blockers
* weekly operating review notes

## Review Checklist

Before using the briefing, check:

* Is today’s focus clear?
* Are key meetings prepared?
* Are decisions visible?
* Are stakeholder sensitivities identified?
* Are follow-ups specific?
* Are risks stated without exaggeration?
* Are assumptions separated from evidence?
* Is sensitive information handled safely?
* Is the recommended first action practical?
* Does the briefing support judgement rather than replace it?

## Output Quality Bar

A useful daily briefing should be:

* concise
* practical
* decision-aware
* priority-led
* sensitive to stakeholders
* realistic about capacity
* clear about follow-ups
* explicit about uncertainty
* safe to store or share only where appropriate

## Failure Modes

This template is not working if:

* it becomes too long to use
* every task is treated as urgent
* the briefing creates more work than clarity
* stakeholder sensitivity is ignored
* admin overwhelms strategic focus
* assumptions are presented as facts
* sensitive information is copied into the repo
* the recommended action is vague
* the user cannot quickly see what matters most
