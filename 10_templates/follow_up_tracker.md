# Follow-Up Tracker

## Purpose

This template helps the Chief of Staff Agent track promised follow-ups, dependencies and open loops without turning them into hidden mental load.

It is designed to help the user:

* capture follow-ups clearly
* distinguish promises from reminders
* identify dependencies
* prevent missed commitments
* prepare stakeholder follow-through
* close loops intentionally
* escalate blocked or sensitive items
* protect trust by making commitments visible

North Star:

> Increase strategic opportunities by helping leaders make better strategic decisions.

## When To Use

Use this template when there are follow-ups from:

* meetings
* stakeholder conversations
* project updates
* decision discussions
* daily briefings
* weekly operating reviews
* inbox triage
* draft artefact reviews
* shipping recommendations
* unresolved dependencies

Use it when a follow-up has enough importance that it should not rely on memory.

Do not use this template for:

* complex project planning
* formal decision records
* sensitive stakeholder commentary
* customer-level records
* confidential organisational source data
* tasks that should be discarded after completion
* issues requiring immediate escalation rather than tracking

## Storage Recommendation

Most follow-up trackers should remain temporary working artefacts. Store only follow-ups that create durable value, decision context, stakeholder pattern evidence, project continuity or career evidence.

| Item | Store?   | Suggested Location | Reason | Sensitivity |
| ---- | -------- | ------------------ | ------ | ----------- |
|      | Yes / No |                    |        |             |

Suggested locations:

* `00_inbox/` for unprocessed notes
* `03_decision_briefs/` for decision-required items
* `05_lessons_learned/` for reusable learning
* `06_stakeholder_patterns/` for reusable stakeholder insight
* `08_projects/` for project-specific follow-ups
* `01_career/` for career evidence
* `10_templates/` for reusable improvements to this tracker

Do not store:

* temporary reminders with no reuse value
* sensitive stakeholder commentary
* raw organisational source data
* customer-level records
* confidential extracts
* duplicated task noise

## Review Checklist

Before using the tracker, check:

* Are promised follow-ups clearly captured?
* Are owners and audiences clear?
* Are due dates or timing expectations visible?
* Are dependencies separated from tasks?
* Are blocked items named?
* Are stakeholder-sensitive items flagged?
* Are decision-linked follow-ups escalated out of the tracker?
* Are completed follow-ups closed or removed?
* Is sensitive information removed or generalised?
* Is the tracker short enough to use?
* Does the tracker reduce mental load rather than create maintenance burden?
* Does the tracker protect trust by making commitments visible?

## Output Quality Bar

A useful follow-up tracker should be:

* practical
* easy to scan
* clear about ownership
* clear about timing
* explicit about dependencies
* sensitive to stakeholder context
* honest about blockers
* safe to store or discard
* focused on closing loops
* subordinate to strategic priorities

## Failure Modes

This template is not working if:

* it becomes a permanent dumping ground
* every reminder is treated as a commitment
* follow-ups are tracked without owners
* sensitive items are treated as routine
* decision dependencies are hidden as tasks
* completed items are never closed
* the tracker becomes harder to maintain than the follow-ups themselves
* low-value follow-ups are repeatedly carried forward
* stakeholder trust risks are missed
* the user cannot quickly see what needs action next
