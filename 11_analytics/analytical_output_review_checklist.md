# Analytical Output Review Checklist

## Purpose

This checklist reviews analytical outputs before they are stored, shared or used in Strategic OS decision-making.

It helps ensure analytical material is source-aware, caveated, proportionate, privacy-safe and decision-relevant.

## Strategic OS North Star

> Increase strategic opportunities by helping leaders make better strategic decisions.

Analytical outputs should improve judgement, not create false certainty.

## When To Use

Use this checklist before analytical outputs are:

- stored in `11_analytics/`
- used in a decision brief
- used in an opportunity review
- used in stakeholder communication
- used in executive-facing material
- used in governance
- used to support a recommendation
- reused by an agent

Use it for:

- analytical summaries
- trend summaries
- anomaly summaries
- source conflict notes
- metric caveat notes
- metric definition notes
- data quality notes
- decision-support evidence
- Genie-supported outputs

## Failure Modes

This checklist is failing if:

- it approves outputs without caveats
- it focuses on formatting rather than decision usefulness
- it misses source uncertainty
- it allows raw data into Strategic OS
- it allows exploratory outputs to become official claims
- it fails to lower confidence when evidence is weak
- it misses human review triggers
- it creates review burden without improving judgement

## Minimal Use Rule

Use this checklist for analytical outputs that may be reused or influence decisions.

Do not apply full review to every temporary analytical query.
