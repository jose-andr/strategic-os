# Strategic OS

## Purpose

Strategic OS is a personal augmentation system designed to help transform complexity into strategic decisions, improve stakeholder alignment, and create more strategic opportunities.

The system supports a Business Design and Human-Centred Design practice through AI-enabled sensemaking, decision support, stakeholder management, analytical interpretation, operating discipline and career development.

## Design Principles

### Decision First

Outputs should be structured around decisions rather than findings.

### Human In The Loop

Humans make decisions.

Agents provide synthesis, recommendations, prompts, analysis, coordination and coaching.

### Privacy By Design

No organisational source data is stored in this repository.

Only personal knowledge, frameworks, observations, caveats, summaries and reusable patterns are retained.

### Employer Independent

Knowledge should remain portable and useful throughout a career.

### Markdown First

All strategic knowledge should be AI-readable and future-proof.

### Ship Over Perfect

Progress, learning and influence are prioritised over unnecessary refinement.

### Evidence With Caveats

Evidence, assumptions, interpretation, recommendation and action should remain distinct.

Analytical outputs should state their source, caveats, confidence and human review requirements.

Components:

- GitHub for reusable knowledge, frameworks and templates

- Make.com for workflow orchestration
- Slack for user interaction
- Strategic OS markdown files for durable, portable knowledge

Strategic OS stores interpretation, not raw organisational data.

## Source Of Truth Rules

Strategic OS does not replace systems of record.

| Information Type | Source Of Truth | Strategic OS Role |
| --- | --- | --- |
| Strategic OS structure | GitHub | Store reusable frameworks, templates and agent guidance |
| Governed data | Databricks or approved data platform | Store interpretation, caveats and decision-support evidence only |
| Reporting | Power BI or approved reporting layer | Store source rules and caveats, not copied reports |
| Organisational documents | SharePoint or OneDrive | Store summaries or reusable patterns only |
| Communication | Teams, email and calendar | Store abstracted follow-up or decision context only |
| Delivery tasks | Jira or official task system | Store summaries and coordination notes only |
| Customer records | Official customer systems | Do not store records in Strategic OS |
